# H. pylori first governed public-layer pass v1

## Status

Entered.

## Function

This note records the first governed public-layer pass for the second case.

It exists because the second case is no longer only:

- an authorized shell,
- an object skeleton,
- and a thin source structure.

It now also has:

- public claim pages,
- first written source pages,
- and a snapshot seed that is already functioning as a release-view surface.

That means the next step cannot just be “add more content.”
It has to become:

> make the current public-facing layers governed enough that they do not silently drift.

---

## One-sentence ruling

Current ruling:

> **The second case is now ready for its first governed public-layer pass.**
> **That pass should begin with claim-page layering, source-page layering, and snapshot release-view layering.**

Do not jump straight to renderer generalization or a page orchestrator.

---

## Why this pass begins here

The second case still does **not** yet have:

- a page surface like the first case,
- a public-layer atlas,
- or a multi-check orchestrator.

That is fine.

What it already has is enough to justify a lighter but real governance move:

- claim pages now carry visible judgment, support, challenge, and lineage,
- source pages now carry visible source identity, role, and object usage,
- snapshot-v0 already acts as a whole-case release-view seed.

These are exactly the kinds of public surfaces that drift if they are left ungoverned.

---

## Current pass scope

This first governed public-layer pass opens only the following work:

1. a claim-page layering rule for `h-pylori-ulcer`
2. a source-page layering rule for `h-pylori-ulcer`
3. a snapshot section layering rule for `h-pylori-ulcer`
4. one checker per layer
5. one workflow per checker

It does **not** yet open:

- page rendering work,
- a public-layer atlas,
- a hidden-checks orchestrator,
- or repository-wide public-layer doctrine.

---

## Practical goal

The practical goal of this pass is narrow:

> make the current second-case public surfaces structurally auditable,
> without pretending the second case has already reached first-case maturity.

---

## Verdict

Current verdict:

> The second case has crossed the threshold where public-facing pages should no longer remain purely informal.
>
> The right next move is a first governed public-layer pass.
>
> That pass should stay local, lightweight, and structural.
