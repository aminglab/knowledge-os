# Snapshot Subsection Semantic Anchors v1

This file defines the current **snapshot subsection semantic anchors v1** for the `h-pylori-ulcer` case.

Its purpose is narrow:

- keep the main semantic commitments inside `snapshot-v0.md` from drifting into vague narrative prose,
- preserve the minimal role of the most important snapshot subsections,
- and give the snapshot layer a checker-friendly semantic anchor surface.

This file is case-scoped.
It does **not** claim to be a repository-wide snapshot doctrine.

---

## Why this layer exists

The current snapshot-section layering rule already protects structure:

- `Title`
- `What this page is`
- `Current visible judgment`
- `Why this case matters`
- `Core claim neighborhood`
- `What changed later`
- `Snapshot reading path`
- `Included objects`

That is necessary.
It is not sufficient.

A snapshot can keep all those headings and still slowly lose the specific semantic commitments that make the case readable as this case rather than as generic medical-reversal prose.

This file exists to stop that drift.

---

## Entry surface

Each subsection anchor entry below exposes at least:

- `Section`
- `Required anchors`

The anchors are intentionally modest.
They do not freeze every sentence of snapshot prose.
They only preserve the minimal phrases that should remain visible if the subsection is still honestly describing its current role.
