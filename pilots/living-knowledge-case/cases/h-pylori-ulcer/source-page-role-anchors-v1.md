# Source Page Role Anchors v1

This file defines the current **source-page role anchors v1** for the `h-pylori-ulcer` case.

Its purpose is narrow:

- keep each source page's `Source role in the case` section from drifting into generic prose,
- preserve the specific role each source plays inside the current case,
- and give the source-page layer a checker-friendly semantic anchor surface.

This file is case-scoped.
It does **not** claim to be a repository-wide source doctrine.

---

## Why this layer exists

The current source-page layering rule already protects structure:

- `Source identity`
- `Source role in the case`
- `Case usage`
- `Object usage`
- `Public reading routes`

That is necessary.
It is not sufficient.

A source page can keep all those headings and still slowly lose the role that makes the source matter in this case.

This file exists to stop that drift.

---

## Entry surface

Each role-anchor entry below exposes at least:

- `Source id`
- `Required role anchors`

The anchors are intentionally modest.
They do not freeze every sentence of source-page prose.
They only preserve the minimal role phrases that should remain visible if the page is still honestly describing the source's case function.
