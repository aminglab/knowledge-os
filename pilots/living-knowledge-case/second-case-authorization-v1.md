# Second Case Authorization v1

## Status

Authorized.

## Function

This document records the first explicit authorization to open a second case inside the **Living Knowledge Case Pilot**.

It exists because the current extraction discipline is not self-activating.
The shell-copy checklist under the first case becomes active only after a second case is explicitly authorized.

This file is that authorization.

>
> Copy the shell.
>

>

## Authorized second case

The authorized second case is:

- [`cases/h-pylori-ulcer/`](./cases/h-pylori-ulcer/)

Working title:

> **H. pylori and Peptic Ulcer Disease: a claim that moved from false rejection to clinical stabilization**

---

## Why this case is being opened now

The second case is being opened now for four reasons.

1. The first case already proved that the pilot can carry a false-elevation lifecycle.
2. The second case should test the opposite direction: false rejection moving toward stabilization.
3. The case is bounded enough to support a first object skeleton without exploding into an encyclopedia.
4. The project now needs a real shell-reuse test rather than more seam-only discussion.

---

## Governing extraction rule

The governing extraction rule remains the same:

> Copy the shell. Rewrite the payload. Do not clone the controversy.

That means:

- structural directories may be reused,
- public-layer expectations may be reused,
- object-envelope expectations may be reused,
- but claims, evidence, dissent, verdict, source pack, and snapshot wording must be rewritten for the new case.
