# COCKPIT2 Object Browse Grammar v1

## Status

Drafted for current threshold.

## Function

This file defines the minimum object-browse grammar for COCKPIT2.
It specifies how a stronger interactive cockpit may lawfully reveal object neighborhoods without changing governed object state.

---

## Governing principle

Object browsing in COCKPIT2 is lawful only when every stronger view remains anchored in named governed objects.
The cockpit may surface neighborhoods more richly, but it may not replace the governed object layer with frontend-only meaning.

---

## Minimum browse modes

### 1. Claim neighborhood browse
The cockpit may foreground a claim together with:

- direct support surface
- direct challenge surface
- lineage placement
- current visible judgment

This mode is lawful only when the visible neighborhood can be traced back to named case objects.

### 2. Evidence adjacency browse
The cockpit may foreground an evidence object together with:

- supported claim adjacency
- source-facing anchors
- route relevance
- current public placement

This mode must remain object-facing, not document-pile browsing.

### 3. Dissent pressure browse
The cockpit may foreground a dissent object together with:

- attack target
- pressure location
- response posture
- current closure status

This mode is intended to make pressure visible, not to collapse dissent into generic comments.

### 4. Verdict boundary browse
The cockpit may foreground a verdict object together with:

- target claim
- current level
- basis surface
- visible upgrade or downgrade boundary

This mode is lawful only when the verdict remains visibly tied to governed basis rather than frontend prose.

### 5. Route browse
The cockpit may foreground a bounded route surface together with:

- route entry point
- route continuation
- route split or pressure zone
- current route status

Route browse does not yet imply repository-wide route generalization.

---

## Browse integrity requirements

Every lawful browse mode must preserve all of the following:

- one visible focus anchor
- one bounded neighborhood definition
- one readable case identity
- no fake object creation
- no hidden mutation

---

## Forbidden shortcuts

COCKPIT2 browse grammar does **not** allow:

- anonymous floating panels with no object anchor
- generic graph exploration detached from case semantics
- mixing pilot objects into one unbounded namespace view
- frontend-only derived verdicts
- frontend-only lineage rewriting

The cockpit may browse more strongly.
It may not invent a second object regime.
