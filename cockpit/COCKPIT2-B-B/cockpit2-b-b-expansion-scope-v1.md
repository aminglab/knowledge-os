# COCKPIT2-B-B Expansion Scope v1

## Status

Drafted for current threshold.

## Function

This file defines the current expansion scope for COCKPIT2-B-B.
It exists to keep the second-stage result-pattern expansion deliberate, narrow, and honest.

---

## Scope rule

COCKPIT2-B-B should widen the structured result-card pattern, but it should still stop short of universal trigger coverage.
The goal is patterned expansion, not immediate completion.

---

## Current expansion scope

The currently preferred expansion set is:

- `scan missing evidence`
- `draft dissent response`

These should join the already proven first-slice triggers:

- `check upgrade conditions`
- `prepare progress summary`

This yields a four-trigger structured result surface while still leaving at least one trigger outside the current coverage boundary.

---

## Why this scope is lawful

`scan missing evidence` naturally supports a bounded `scan_result` card.
`draft dissent response` naturally supports a bounded `draft_response` card.

Together with the already proven `condition_check` and `summary_draft` cards, this gives the cockpit a real result-pattern expansion without pretending that the trigger universe is now complete.

---

## Current non-scope

The current lift should not yet require `suggest canonical continuation` to produce a full structured card.
That broader move may come later after the expanded result pattern proves coherent.
