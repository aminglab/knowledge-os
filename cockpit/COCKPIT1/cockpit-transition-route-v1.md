# COCKPIT1 Transition Route v1

## Status

Drafted for construction.

## Function

This file defines the lawful route from **Founding Session** to **Object Freeze** to **COCKPIT1**.

---

## One-sentence transition ruling

> **No user should be dropped into the cockpit from pure chaos.**
>
> **COCKPIT entry becomes lawful only after a first object floor and route map have formed.**

---

## Three-mode route

### 1. `founding_mode`
Used when the project still has:

- unstable terminology
- no clear object set
- no claim floor
- no usable route map

Expected outputs:

- project charter draft
- candidate objects
- initial claims
- open risks
- route fragments

### 2. `structuring_mode`
Used when the project has enough material to compress into governed shape.

Expected outputs:

- first object table
- first route tree
- claim candidates
- evidence buckets
- dissent buckets
- freeze candidates

### 3. `cockpit_mode`
Used only after the following minimum freeze threshold:

- at least one named project or route
- at least one stable claim object
- at least one evidence or dissent surface
- a visible current-state grammar
- a current focus object that can anchor actions

---

## Freeze threshold

Entry into COCKPIT1 requires:

- `PASS_OBJECT_FLOOR_FORMED`
- `PASS_ROUTE_MAP_PRESENT`
- `PASS_ACTION_ANCHOR_PRESENT`

If any of these fail, the user remains in founding or structuring mode.

---

## Current project mapping

For this repository, the current pilot work implies that:

- the object floor already exists
- live case routes already exist
- public-layer governance already exists
- therefore cockpit construction may start without pretending that founding mode is complete for all future projects

---

## Final transition block

- route stages: 3
- lawful cockpit entry: freeze threshold passed
- cockpit entry from pure free chat: forbidden
