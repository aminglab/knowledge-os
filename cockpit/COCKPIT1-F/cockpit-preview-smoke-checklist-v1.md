# COCKPIT1-F Preview Smoke Checklist v1

## Status

Drafted for use.

## Function

This file defines the minimum smoke checklist for a lawful cockpit prototype preview run.

---

## Checklist

### A. Preview launch
- [ ] `preview_cockpit.py` starts without immediate failure
- [ ] local preview URL is printed
- [ ] browser opens, or `--no-browser` behavior is respected

### B. Shared frame
- [ ] left rail is visible
- [ ] main panel is visible
- [ ] right rail is visible
- [ ] sidecar is visible and subordinate

### C. State sequence
- [ ] landing state is reachable
- [ ] `h-pylori-ulcer` claim focus is reachable
- [ ] `h-pylori-ulcer` verdict boundary is reachable
- [ ] `power-posing` dissent pressure is reachable
- [ ] `power-posing` route map is reachable

### D. Honesty checks
- [ ] the prototype is described as static and bounded
- [ ] the operator does not present it as a production frontend
- [ ] the operator does not present it as a write-capable workspace

---

## Final checklist block

- launch checks: 3
- frame checks: 4
- state checks: 5
- honesty checks: 3
