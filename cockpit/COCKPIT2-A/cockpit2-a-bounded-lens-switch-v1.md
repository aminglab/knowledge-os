# COCKPIT2-A Bounded Lens Switch v1

## Status

Drafted for current threshold.

## Function

This file defines the minimum lawful lens-switch behavior for COCKPIT2-A.
The lens switch is the first deliberate interaction upgrade beyond the COCKPIT1 static floor.

---

## Governing principle

A lens switch changes foreground emphasis only.
It does not:

- mutate governed objects
- create new verdicts
- create new lineage
- replace case semantics with frontend-only prose

---

## Minimum lawful lens set

COCKPIT2-A uses the following bounded lenses:

- `focus`
- `pressure`
- `route`
- `boundary`

### focus
Foreground the current object and its nearest governed neighborhood.

### pressure
Foreground the strongest current support/challenge pressure around the current focus.

### route
Foreground continuation, route plurality, or public-surface route placement.

### boundary
Foreground retained holds, verdict limits, and non-upgrade honesty.

---

## Lens integrity rules

Every lawful lens switch must preserve:

- one visible state identity
- one visible case identity
- one visible active object anchor
- one visible non-mutation posture

A lens may alter emphasis.
It may not alter what the governed object currently is.

---

## Forbidden drift

The lens switch becomes unlawful if it starts behaving like:

- hidden object transformation
- hidden verdict transformation
- free graph wandering detached from case grammar
- runtime task control disguised as browsing

---

## Acceptance note

The COCKPIT2-A prototype does not need a large lens universe.
It only needs to prove that one bounded lens-switch surface can exist without breaking object honesty.
