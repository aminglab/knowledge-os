# COCKPIT2-B-A Result-Card Lift v1

## Status

Drafted for current threshold.

## Function

This file defines the first concrete result-card lift for COCKPIT2-B-A.
It exists to prove that a bounded cockpit trigger can render a richer draft output surface than an audit note alone.

---

## Governing principle

The first result-card lift is lawful only if the output remains:

- draft-only
- bounded by output class
- tied to a named object anchor
- visibly narrower than mutation

---

## Minimum implemented lift

The first lift should implement at least one structured result card for one bounded trigger.
A stronger but still lawful initial implementation may support more than one trigger, but one is the floor.

---

## Minimum result-card sections

The first structured result card should visibly expose:

- title
- output class badge
- anchor reference
- draft-only honesty marker
- summary body
- basis hints or evidence note
- unresolved remainder
- next-step hint
- audit trace note

---

## Required UI reading

The right reading is:

> cockpit-native draft result surface.

The wrong reading is:

> authoritative action result.

---

## Current boundary

The first result-card lift may make trigger outputs easier to read.
It may not make them more authoritative.
