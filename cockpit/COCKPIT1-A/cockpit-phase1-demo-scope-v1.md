# COCKPIT1-A Phase 1 Demo Scope v1

## Status

Drafted for scoping.

## Function

This file defines the bounded demo scope for COCKPIT1-A.

---

## One-sentence scope ruling

> **COCKPIT1-A Phase 1 should demonstrate real case binding, not visual completeness.**

---

## Required demo contents

The Phase 1 demo must include:

- one landing state
- one claim focus state
- one dissent pressure state
- one verdict boundary state
- one route / map state

At least two distinct real cases must appear across those states.

---

## Primary demo sequence

1. Landing screen with both pilot cases visible
2. `h-pylori-ulcer` claim focus screen
3. `h-pylori-ulcer` verdict boundary screen
4. `power-posing` dissent pressure screen
5. `power-posing` route / map screen

---

## Explicit non-goals

Phase 1 demo does not require:

- polished visual design
- responsive production UI
- multi-user collaboration
- write-capable runtime
- live model routing
- public release export integration

---

## Pass condition

Phase 1 demo is sufficient if it proves that:

- the cockpit can center on real governed objects
- the cockpit can switch between two non-identical cases
- the object views do not collapse into generic prose cards
- the sidecar remains visibly subordinate

---

## Final scope block

- minimum screens: 5
- minimum bound cases: 2
- polish requirement: none
- proof target: lawful case-to-cockpit binding
