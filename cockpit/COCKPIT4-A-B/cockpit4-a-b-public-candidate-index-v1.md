## Status

Drafted for current threshold.

## Function

This file documents the first public-candidate index floor.
It pairs one emitted COCKPIT4-A-A candidate sample with a compact registry surface under:

- `cockpit/COCKPIT4-A-B/public-candidate-index-v1.json`

---

## Indexed sample

The current index contains one candidate:

- `public-candidate:h-pylori-ulcer-summary:v1`

---

## Required registry fields

Each registry entry must expose at least:

1. `object_id`
2. `candidate_stage`
3. `source_object_kind`
4. `candidate_file`
5. `dissent_residue_status_class`
6. `snapshot_id`
7. `registry_status`

---

## Current lawful reading

The right reading is:

> this index makes public candidates discoverable and auditable.

The wrong reading is:

> this index is a public publication catalogue.

---

## Boundary

Indexing a public candidate does not create publication authority, verdict finalization, governed mutation authority, or public-truth status.
