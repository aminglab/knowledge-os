# COCKPIT4-A Entry Decision v1

## Status

Entered at current threshold.

## Function

This file records the entry decision for **COCKPIT4-A / Public Candidate Admission Floor v1**.

COCKPIT3-B has already been formally closed as a review-boundary formation stage.
The next lawful move is not to widen private cockpit complexity further.
It is to define the minimum admission object that would let a governed private result become a **public candidate object** without implying publication authority.

---

## One-sentence entry verdict

Current entry verdict:

> **ENTER_COCKPIT4_A_PUBLIC_CANDIDATE_ADMISSION_FLOOR**
>
> The project may now open COCKPIT4-A because the private cockpit line has already formed its review-boundary and family-label discipline, and the next lawful threshold is to define what minimum object grammar would be required before any private governed surface could be admitted into a public-candidate layer while still remaining non-published and non-authoritative.

---

## Immediate purpose

COCKPIT4-A exists to do four things:

1. define the minimum object grammar for a `public_candidate_object`;
2. define the admissibility fields that must travel with any such candidate;
3. preserve the separation between candidate admission and publication authority;
4. define the acceptance grammar for a lawful public-candidate admission floor.

---

## Boundary

COCKPIT4-A does **not**:

- authorize publication
- authorize verdict finalization
- authorize governed mutation
- claim that a public candidate is already public truth
- replace the existing private cockpit governance line

It only opens the admission floor for a possible public-candidate layer.
